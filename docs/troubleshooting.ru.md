# Диагностика BRKCHRD 0.4.0 — Русский

## BRKCHRD не появился в Ports

```bash
ssh trimui 'ls -l "/userdata/roms/ports/BRKCHRD.sh" /userdata/roms/ports/brkchrd/*.aarch64'
ssh trimui 'chmod +x "/userdata/roms/ports/BRKCHRD.sh" /userdata/roms/ports/brkchrd/*.aarch64 && sync'
```

Обнови список игр или перезагрузи консоль.

## Программа сразу закрывается

```bash
ssh trimui 'cat /userdata/roms/ports/brkchrd/conf/brkchrd.log'
```

Типовые причины: отсутствует бинарник, неправильная архитектура, ошибка SDL audio/video или неполное окружение PortMaster.

## L1 работает как L2 или R1 как R2

Открой Settings и переключи:

- `SWAP L1/L2`
- `SWAP R1/R2`

Подтверждённая раскладка отдаёт четыре задние клавиши раздельно, но другая версия Knulli может поменять физический порядок внутри стороны. SWAP действует только в BRKCHRD и безопаснее глобального переназначения.

Ожидаемые роли v0.4:

```text
left/right shoulder = передние октавные клавиши
left stick button   = L1
left trigger        = L2
right stick button  = R1
right trigger       = R2
```

Первые строки лога:

```bash
ssh trimui 'sed -n "1,160p" /userdata/roms/ports/brkchrd/conf/brkchrd.log'
```

Ищи `mapping:`, `roles:`, `controller button down:`, `raw button down:` и `raw axis:`.

## L2 не меняет режим D-pad

L2 переключает режим только по новому нажатию и при закрытых Settings. В шапке должен быть цикл:

```text
CHORD → SOUND → PERF FX
```

Если удержание нужной физической кнопки вместо этого меняет слой слева, включи `SWAP L1/L2`.

## В SOUND пропала выбранная окраска

SOUND не должен сбрасывать окраску. Вернись в CHORD и посмотри строку ACTIVE. Сброс происходит только при повторном нажатии выбранного направления в отображаемой палитре.

Само удержание L1 только показывает другую палитру и ничего не выбирает.

## PERF FX завис после отпускания

Обычные FX должны восстанавливаться при:

- возврате D-pad в центр;
- переключении режима по L2;
- открытии Settings;
- нормальном выходе.

Если эффект остаётся, приложи `brkchrd.log` и укажи направление D-pad. Удержание Start остановит ноты; открытие Settings принудительно вернёт обычные эффекты.

## PERF FX звучит не буквально как название

Названия v0.4 описывают музыкальный жест. Reverse-ish и Shimmer-ish пока являются сочетаниями Delay/Reverb/Crusher/Chorus, а не отдельными reverse-buffer или pitch-shimmer алгоритмами. Это известное ограничение DSP, а не ошибка кнопок.

## Перепутаны ABXY или позиции

Ожидаемая физическая схема:

```text
        X
    Y       A
        B
```

Внутри SDL Knulli может называть нижнюю B как A, правую A как B, левую Y как X и верхнюю X как Y. BRKCHRD переводит позиции. Если подсвечивается не та физическая позиция, нужен лог другой mapping-конфигурации.

## Как вернуть BASE

В CHORD покажи палитру, в которой была выбрана окраска, и повторно нажми направление. Если окраска выбрана из L1-палитры, удерживай L1 во время повторного нажатия.

## Как остановить все ноты

Удерживай Start около 0,85 секунды. Latch в 0.4 удалён.

## Треск или высокая нагрузка

- сначала поставь обычные FX1/FX2 в Off;
- не сочетай максимальный Release с плотными расширенными аккордами при первом тесте;
- сравни обычную работу и удержание PERF FX;
- проверь UI Motion = Off;
- сравни динамик и наушники;
- запиши движок, окраску, банк, режим игры, направление D-pad и нагрузку CPU.

## Сброс настроек

```bash
ssh trimui 'mv /userdata/roms/ports/brkchrd/conf/brkchrd.cfg /userdata/roms/ports/brkchrd/conf/brkchrd.cfg.bak'
```

Следующий запуск использует заводские настройки.

## Нормальный баг-репорт

Нужны: версия Knulli, имя архива, `brkchrd.log`, физическая кнопка и ожидаемая роль, режим D-pad, состояние L1/R1/R2, сохранённая окраска, обычные FX, динамик/наушники, задержка, треск, нагрузка и фото/видео для проблем интерфейса или управления.
