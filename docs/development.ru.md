# Разработка BRKCHRD — Русский

## Структура проекта

```text
include/brkchrd/       публичные C++-интерфейсы
src/music.cpp          построение аккордов и раскладок
src/synth.cpp          голоса, модели синтеза и эффекты
src/sdl_main.cpp       SDL-ввод, интерфейс, конфиг и аудио
tests/test_main.cpp    тесты гармонии, раскладок и звука
tools/render_demo.cpp  офлайн-рендер пресетов
tools/package_*.sh     упаковка PortMaster и NextUI
packaging/             launcher, метаданные и графика
assets/branding/       айдентика проекта
```

## Сборка

Нужны CMake 3.16+, компилятор C++20 и SDL2 development files.

```bash
sudo apt-get install build-essential cmake pkg-config libsdl2-dev zip file
cmake -S . -B build -DCMAKE_BUILD_TYPE=RelWithDebInfo
cmake --build build -j2
ctest --test-dir build --output-on-failure
```

Запуск на компьютере:

```bash
./build/brkchrd-sdl
```

Headless-проверка:

```bash
BRKCHRD_SKIP_SPLASH=1 SDL_VIDEODRIVER=dummy SDL_AUDIODRIVER=dummy \
timeout 3s ./build/brkchrd-sdl
```

## Инварианты гармонической логики

Изменения `music.cpp` должны сохранять:

- отсортированные интервалы без дублей;
- нулевой интервал как гармоническую тонику;
- качество минорных и уменьшённых ступеней;
- независимость стабильной раскладки от предыдущего аккорда;
- точный сдвиг каждого голоса на 12 полутонов при смене октавы;
- привязку живого Voice Lead к выбранной октаве;
- безопасные низкие раскладки для Bass и Heavy.

Для каждого воспроизводимого отчёта с реального устройства нужен регрессионный тест.

## Правила аудиодвижка

У целевой консоли ограниченный CPU и маленький динамик. Предпочтительны ограниченное число голосов, общая модуляция, устойчивые фильтры, явное ограничение параметров и компенсация уровня для каждой модели. В посэмпловой обработке не должно появляться лишней работы с памятью или файлами.

Проверяйте звук на наушниках, линейном выходе и встроенном динамике.

## Правила интерфейса

Логическое разрешение — 512×384, шрифт встроенный bitmap.

- текст и значения остаются внутри рамок;
- правые значения выводятся через `text_right` с отступом;
- шкала выбранного параметра остаётся видимой;
- длинные строки проверяются на самом длинном заводском названии;
- важная сценическая информация не зависит только от слабого различия цветов;
- подписи соответствуют физическим кнопкам Brick.

## Упаковка

PortMaster:

```bash
sh tools/package_portmaster.sh build-aarch64 dist
```

NextUI:

```bash
sh tools/package_nextui.sh build-aarch64 dist
```

Оба пакета содержат один бинарник, документацию, GPL, `NOTICE.md` и сторонние уведомления. Launcher различаются путями и хранением постоянных данных.

## Подготовка релиза

Обновите `kVersion`, версию CMake, имена демо и архивов, README, документацию, `pak.json`, метаданные платформ и проверки CI. Тег публикуется только после Linux-тестов, SDL smoke test, AArch64-тестов и проверки обоих пакетов.

## Вклады в проект

Изменения должны быть сфокусированными и сопровождаться описанием проверки. Принятые вклады распространяются по GPL-3.0-or-later, если заранее не согласовано иное. Производные версии сохраняют уведомление из `NOTICE.md`.
